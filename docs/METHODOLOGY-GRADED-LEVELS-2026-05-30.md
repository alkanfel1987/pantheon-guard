# Методика выявления манипуляции — градуированные уровни (v0.1)

Статус: рабочий артефакт Яджни (внутри `pantheon-guard`, не внешний). При выходе во
внешний мир (КП / Foundation / публичная методика) — через `/quarantine` (CLAUDE.md §5).
Дата: 2026-05-30. Происхождение: со-дизайн с пользователем 2026-05-30, якорь — `L2-CONCLUSION.md`.

## 1. Ядро: манипуляция = разрыв между обещанием и payload

Манипулятивность **не определяется формой заголовка**. Эмпирический потолок измерен:
регекс на поверхности даёт fresh catch 8.3%; `L2-CONCLUSION` — «заголовок ~10 слов,
модель не учит механизм; нужно тело, чтобы проверить, withheld ли payload».

**Определение.** Текст манипулятивен, если при проходе по цепочке (заголовок → тело →
оффер → следующий слой) **обещание поверхности предаётся**: payload не доставлен,
bait-and-switch, экстрактивная воронка, обрыв источника, эксплуатация уязвимости без
доставки ценности.

Та же форма без предательства обещания = **шаблонное поведение без намерения**
(листикл, ведущий к реальному списку; «почему X», ведущий к честному объяснению).

Разделитель — **разрешение цепочки**, не лексика поверхности.

## 2. Три уровня

| Уровень | Определение | Где устанавливается | Вердикт? |
|---|---|---|---|
| **L0 — шаблон / легитимно** | форма манип-типа есть, цепочка доставляет обещанное честно | только после прохода цепочки | да (оправдание) |
| **L1 — возможная манипуляция** | поверхностный сигнал есть, цепочка не пройдена / неоднозначна | layer-1 (заголовок) | **нет — флаг, не приговор** |
| **L2 — однозначно манипуляция** | проход цепочки подтвердил разрыв обещание↔payload | layer-2+ (глубина) | да (обвинение) |

**Принцип честности (фиксирует роль детектора):** поверхностный детектор физически
может выдавать **только L1**. Выдавать L1 как L2 — источник всех FP проекта
(healthcare флагал медnovости, news флагал легитимное «sources say»). Поверхность даёт
флаг; вердикт «однозначно» даёт только глубина.

## 3. Глубина зависит от среды

| Среда | Слои | Где вскрывается «однозначно» |
|---|---|---|
| Новости | 2 (заголовок → статья) | layer-2 сразу |
| Соцсети / маркетинг | N (хук → «ссылка в шапке» → канал → курс → апселл) | глубже; цепочку надо пройти дальше |
| Скам / воронка | N+ (часто за пейволлом / гейтом) | самый дорогой проход |

## 4. Операционная архитектура (следствие)

Проход цепочки **дорогой** (фетч ссылок, чтение тел, пейволлы). Отсюда двухтактность:
1. **Дёшево, на потоке:** поверхностный флаг **L1 возможная** (регекс/семантика — их честная роль).
2. **Дорого, только на флагнутых:** проход глубины → подтверждение/снятие до **L0 или L2**.

**Краудсорс глубины:** callout-меньшинство, прошедшее воронку и крикнувшее «там пусто/
развод», уже выполнило проход цепочки. Их комментарии — дешёвый сигнал разрешения цепочки
(кандидатные метки L2/L0, финал — адъюдикация человека).

## 5. Следствие для валидации

Критерий завершения методики — **не catch детектора**, а согласованность применения
уровней людьми: два разметчика на held-out наборе **цепочек** (не заголовков) дают
Cohen's κ ≥ 0.6 по трёхуровневой шкале. Детектор — лишь одна автоматизация L1-флага.

## 5a. Уточнения из первого живого прохода (bravo, 2026-05-30)

**Worked example #5 (reel bravo.konstantin):**
- Layer 1 хук: «ты убиваешь трафик, не используя LSI… напиши РОСТ — дам файл».
- Layer 2 (профиль): bio «научу создавать и умножать ваш…», воронка лид-магнит→коучинг.
- Разрешение: **L0 по оси «воронка»** (лид-магнит→коучинг — нормальная бизнес-модель) +
  **L2 по оси «истинность»** (хук на ложной посылке: LSI-как-фактор-ранжирования развенчан).

Два уточнения схемы:
1. **Уровень привязан к ОСИ/утверждению, не ко всему посту.** Один пост = L0 по форме И
   L2 по ложной посылке. Вердикт обязан указывать «L2 **на чём**».
2. **Стоимость прохода реальна и иногда неоплатима.** #7 mr.love — профиль режет бота,
   флаг честно остаётся **L1**. Это граница дешёвого слоя, не дефект.

**ahimsa-оговорка:** анализируется риторический паттерн поста, НЕ человек. Реальных людей
в L2 без полного прохода + адъюдикации не записываем (mahā-vrata: satya/ahimsa).

## 5b. Принцип прохода слоёв — только легитимный доступ

«Пробивать защиту» платформ — **запрещено** (asteya/satya + право/ToS + само-противоречие
для этического продукта). Допустимые способы углубления:
- **Аутентифицированная сессия, которой владеет пользователь** (его залогиненный браузер,
  Claude in Chrome) — это не обход, это реальный пользователь смотрит публичный контент.
- **Официальные API / oEmbed**, веб-архивы (Wayback) для лендингов.
- **Ручной проход** пользователем + вставка найденного.
- **Краудсорс глубины** — callout-комментарии тех, кто уже прошёл воронку.
Запрещено: обход пейволлов/анти-бота через evasion, фейк-аккаунты для скрейпа против ToS,
обход access-control. «Однозначно»-вердикт дорогой и легитимный — это фича, не баг.
