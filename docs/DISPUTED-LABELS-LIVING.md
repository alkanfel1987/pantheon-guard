# Реестр спорных разметок — живой набор (living corpus)

Рабочий накопитель пограничных кейсов из живого набора, где метка `catch`/`pass`
под вопросом. Не трогает пред-регистрацию N=509. Когда наберём достаточно —
выводим единой таблицей и классифицируем разом, затем фиксируем решения
(перелейбл + при необходимости новый детектор).

## Принятый критерий (от 2026-06-07)

> **Удержанная развязка** (заголовок прячет ответ → надо идти в статью) = манипуляция (`catch`).
> **Развязка в самом заголовке** (вопрос задан и сразу отвечен) = норм (`pass`), самодостаточная мини-история / юмор.

Открытые оси (по ним ещё нет вердикта — копим примеры):
- **Оценочная рамка** (moralizing): навязанное осуждение через ярлык («Entitled», «I'm Rich»). Манипуляция или нет?
- **Эмоциональная накрутка реального события**: драма-слова («Chaos», «Wild Rampage», «Breaks Down») поверх настоящей новости. Манипуляция или нет?

## Следственный протокол (от 2026-06-07)

Манипуляцию разбираем как состав преступления: чтобы вынести `catch`, нужно
доказать состав целиком, а не «выглядит подозрительно». Пять шагов на кейс:

1. **Опрос по 5W1H** (Кто / Что / Где / Когда / Почему / Как). Чистый
   заголовок отвечает на ключевые W. Манипуляция **намеренно удерживает** один
   (обычно «Что» или «Почему») — это и есть разрыв любопытства.
2. **Сбор улик (принцип Локара — «всякий контакт оставляет след»).** Выписать
   языковые следы: слово-заглушка (take/boast/reason/thing), указательное
   (these/this) без референта, глагол раскрытия (reveals/emerges/revealed),
   оценочный ярлык (Entitled), усилитель эмоции (Chaos/Rampage).
3. **Мотив.** Чего добивается заголовок: клик / возмущение / репост.
4. **Проверка алиби (нулевая версия).** Попытаться оправдать: «это просто
   полный факт, поданный нейтрально». Если все W на месте и тон нейтральный —
   **алиби в силе → `pass`**, даже если тема драматичная. Алиби рушит только
   доказанный состав (удержание / навязанная оценка).
5. **Вердикт + доказательная цепь.** `catch` только при полном составе; в
   вердикте цитируем конкретные улики (как уже делает `packEvidence`).

Презумпция: **при недоказанном составе — `pass`** (как «невиновен, пока не
доказано»). Это и есть анти-FP-дисциплина.

## Тест инсценировки (Артхашастра 4.7.11) на двух открытых осях

Тест: совпадают ли «поверхность» (нейтральный полный факт) и «следы» (то, что
добавлено)? Несоответствие = инсценированная поверхность → `catch`; полный факт
с нейтральным тоном → алиби → `pass`.

## ✅ Вердикты по осям приняты (2026-06-07) — становятся правилами детектора

### Правило A — накрутка (единый принцип «усиление обязано нести доказательство»)

Совпадает с тестом происхождения āgama (улика без источника подозрительна).
- **Количественная ветвь:** величина превышает реалистичный базовый уровень в
  **>1.5× без приведённого доказательства** → `catch`. (Требует базового уровня
  по сфере — будущая возможность; источник: правило пользователя 2026-06-07:
  «реальный результат хочется подкрепить, как достигнут».)
- **Качественная ветвь:** усилитель **не подкреплён** конкретным фактом в самом
  заголовке → `catch`; **подкреплён** → `pass`. (Реализуемо сейчас структурно.)

→ **#279 ЗАКРЫТ → `pass`** (перелейблено): «Chaos/Wild Rampage» подкреплены
фактом «punches and bites flight attendant» → доказательство на месте.

### Правило B — слой доказанности (объединяет moralizing + презумпцию)

Поправка пользователя 2026-06-07: «обвинён ≠ виновен». Это **один принцип,
ключённый на слой доказанности**, а не две оси:

> Вердикт / обвинение / характеристику можно подавать как факт **только если
> доказанность установлена**. Доказано (осуждён, подтверждено) → писать можно.
> Не доказано, но подаётся как факт (или через ярлык) → манипуляция.

Стержень тот же, что āgama/yathādṛṣṭam: **утверждение обязано нести своё
доказательство**. Артхашастра 4.8.06 («обвинение невиновного само карается») —
не отдельная ось, а тот же принцип на двух уровнях:
- **уровень контента:** заголовок пред-осуждает недоказанного → `catch`;
- **уровень детектора:** детектор не «осуждает» чистый текст без доказательства →
  строгий FP-гейт.

**Детектируемый сигнал:** утверждение вины/вердикта/характеристики (ярлык:
«Entitled», «greedy», «monster»; или прямое «guilty») + маркер установления
только на уровне обвинения (charged / accused / alleged / suspected), без маркера
доказанности (convicted / found guilty / sentenced / pleaded guilty) → `catch`.

→ **#278 ЗАКРЫТ → `catch`**: «Officially Charged» = только обвинение; «Entitled»
+ «I'm Rich» = вердикт, поданный как факт до доказательства. Настоящий FN — цель
детектора «слой доказанности».

Остаточная под-ось (не усложняем сейчас): моральные эпитеты даже к доказанно
виновному (осуждение факта vs навешивание характера) — пометить, отложить.
