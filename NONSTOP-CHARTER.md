# NONSTOP CHARTER — pantheon-guard

Запущено 2026-05-30. Ветка: `nonstop/2026-05-30-triage-merge`.
Режим: автономный прогон до **Ступени 1** или жёсткого стопа. Согласовано пользователем «да погнали».

## Цель проекта (методология-центр)
Разгрести незакоммиченный завал → привести репо в честное состояние → собрать held-out корпус и probe → довести до **внешне-проверяемого вердикта** по слиянию «старый регекс-формат + новый semantic (zero-shot NLI)».

## Критерий завершения «done» — лестница внешней валидации
| Ступень | Порог | В прогоне? |
|---|---|---|
| 0. Internal green | тесты 403/403, честные scaffold-метки | старт, НЕ done |
| **1. Held-out + адъюдикация пользователя** | **N ≥ 30 pos + 30 neg, catch Wilson-LB ≥ 50%, FP Wilson-UB ≤ 5%** | **ЦЕЛЬ** (корпус собираю я, вердикт — очередь) |
| 2. 2-й аннотатор + Cohen's κ ≥ 0.6 | цитируемо как Tier 2 | вне прогона (нужен 2-й человек) |
| 3. Рыночный сигнал | ≥1 платящий пилот / ответ целевой компании | вне прогона |

**«done» ≠ «успех».** Завершение = вердикт достигнут, **положительный ИЛИ отрицательный**. Пороги не крутить под «успех» (Я-3, R4). Негативный held-out = валидное завершение (как L2-CONCLUSION).

## Граница автономии
- **АВТО (качу без остановки):** триаж, коммиты (через ворота честности), рефакторинг, докрепы, фиксы тестов, сборка/майнинг корпуса, probe-харнесс, semantic C2/C3. Ворота: тесты+линт зелёные, git-тег на форке.
- **ОЧЕРЕДЬ (готовлю и паркую пользователю, НЕ решаю):** финальная адъюдикация меток, вердикт «работает ли детекция», любое внешнее число, любая отправка outreach. Очередь — в `NONSTOP-LOG.md`.

## Протокол форка
Каждый обратимый форк → `simulate-fork` (forecast + customer-pain + maha-vrata + reversibility + falsifier + premise-check). Выбор фиксируется git-тегом `fork/<имя>/<вариант>` → откат в один `git checkout`. Уверенность <70% → не решаю, в очередь.

## Жёсткий стоп
